# Yoke 0.1.1+launch.19

This release makes browser approval the only fresh machine-authorization path
for Yoke Cloud onboarding.

- `yoke onboard --connect https://app.upyoke.com` now opens hosted environment
  selection and browser approval instead of the team-server token screen.
- Fresh Yoke Cloud runs reject positional tokens, token files, token stdin,
  JSON mode, and non-interactive mode with a direct browser-approval
  instruction, closing neighboring flag-driven bypasses.
- Existing browser-approved Cloud connections and resumed onboarding runs may
  reuse their owner-only credential file. Explicit token entry remains
  available for self-hosted team servers.

Use the normal update path. No schema migration is required.
