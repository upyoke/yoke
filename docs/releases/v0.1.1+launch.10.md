# Yoke 0.1.1+launch.10

This release makes fresh and fleet-upgraded schemas compatible for hosted
universe imports without weakening structural validation.

- Fresh universes now declare GitHub sync receipt columns in the same order
  produced by the historical additive fleet migration.
- Portable schema comparison now ignores physical column order across every
  table because archive restore maps values by column name.
- Column names, types, defaults, nullability, constraints, indexes, triggers,
  policies, views, functions, sequences, and storage properties remain exact.
- A regression test compares complete fresh and upgraded Postgres schema
  fingerprints, and a second proves only portable comparison ignores order.

No existing tenant table is rebuilt or reordered. Fleet-upgraded tenants keep
their current schema and become valid targets for the same portable archives.
