# Yoke 0.1.1+launch.6

This release fixes the hosted browser sign-in introduced in launch.5.

- Normal `authorization_pending` responses now keep polling until the user
  approves the machine instead of being mistaken for a completed response
  without a token.
- Malformed or unexpected polling responses still fail closed.
- Hosted sign-in retry guidance starts a fresh browser authorization and no
  longer suggests pasting an API token.

The normal installer and upgrade paths require no manual migration.
