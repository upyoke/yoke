# Yoke 0.1.1+launch.11

This release hardens production database and ingress behavior discovered while
moving standalone universes into hosted tenants.

- Managed AWS database-secret connections now recover across credential
  rotation by refreshing the current version, then trying the previous version
  only after PostgreSQL reports an invalid password.
- Explicit database bindings and non-password connection failures retain their
  existing fail-loud behavior.
- Standalone service ingress now accepts the bounded strategy-corpus request
  size used by full strategy ingest.
- Process-scoped path claims now derive project authorization from their owning
  work claim, so sanctioned cross-project operations no longer inherit an
  unrelated ambient project.
