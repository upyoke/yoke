# Yoke 0.1.1+launch.12

This release restores full parity between live item/project fields and the
schema created for fresh or fleet-upgraded universes.

- Fresh and upgraded universes now include the three durable item-resolution
  fields used by cancellation and dependency-resolution workflows.
- Project breakage policy is now converged explicitly so governed-migration
  posture survives universe export and import.
- Portable round-trip coverage proves non-empty resolution and policy values
  are preserved.
- The production archive catalog now matches a fresh deployed schema across
  all 78 tables, every COPY column set, and every sequence.
