# Yoke 0.1.1+launch.41

## Highlights

- Adds the canonical one-file universe archive used to move a universe between
  self-hosted and hosted installations.
- Carries the source-freeze receipt inside that archive so an import can prove
  which source state it replaces.
- Makes export, validation, and import share one archive contract across the
  CLI and engine.
- Removes the retired multi-artifact export path in favor of the single
  portable archive.

## Upgrade notes

Hosted Platform services that consume universe portability must advance their
pinned Yoke release to `0.1.1+launch.41` before enabling the matching Platform
surface.

## Compatibility

The archive consumer and producer must both use the one-file universe archive
contract shipped in this release.
