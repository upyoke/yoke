# Yoke 0.1.1+launch.18

This release makes harness-session lifecycle ownership follow the active Yoke
transport instead of assuming a machine-local control plane.

- Hosted clients begin sessions through the registered `sessions.begin`
  function and the matching `yoke sessions begin` adapter, so HTTPS-connected
  workspaces keep session authority on the server.
- Non-production local Postgres universes register, heartbeat, and end harness
  sessions in-process through the hook runner. Machines without a bound local
  universe remain lint-only, and hook delivery stays fail-open.
- Session and claim operations now carry a registry-driven transport-parity
  check so future client/server routing drift fails during verification.

Use the normal update path after the hosted server has been rolled to this
release. Local universes receive the client and engine changes together.
