# Yoke 0.1.1+launch.26

This release fixes hosted GitHub App lifecycle delivery for tenant universes.

- Installation and repository lifecycle authorization now follows the exact
  project named by the verified hosted delivery instead of requiring a
  project literally named `yoke` in every tenant universe.
- The hosted service-token guard remains mandatory, and project authorization
  is checked against the binding being changed.
- Registry coverage now prevents service-only hosted functions from silently
  inheriting literal-Yoke control-plane authority in future releases.

Use the public installer again to update an existing packaged CLI. No schema
migration is required.
