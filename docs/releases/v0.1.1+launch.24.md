# Yoke 0.1.1+launch.24

This release makes the engine distribution the sole runtime version authority.

- Project install, template, and source-link bundles now report the installed
  `yoke-core` version instead of querying a nonexistent `yoke` distribution
  and falling back to `0.1.0`.
- The API application metadata uses the same resolved engine version as the
  health and response-header handshake.
- Source runs without distribution metadata share one explicit unresolved
  fallback constant; released wheels no longer emit misleading bundle
  versions.

Use the public installer again to update an existing packaged CLI. No schema
migration is required.
