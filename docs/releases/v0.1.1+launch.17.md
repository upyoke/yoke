# Yoke 0.1.1+launch.17

This release makes governed database migrations portable across standalone and
hosted universes and completes the canonical-schema cleanup begun in launch.16.

- Governed migration manifests can carry immutable packaged module sources, so
  hosted tenant runtimes apply the same reviewed migration bytes as standalone
  environments.
- The authority validation-copy helper keeps database credentials out of
  process arguments and restores into a clean disposable schema before a live
  migration is rehearsed.
- The fully applied design-document and function-call-ledger one-time migration
  code has been removed after authoritative backup and fingerprint evidence.

Normal clients can use the standard update path. Hosted migration execution
requires the running platform service and the selected product source to resolve
to this same immutable commit.
