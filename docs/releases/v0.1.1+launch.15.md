# Yoke 0.1.1+launch.15

This release consolidates item design specifications into the canonical item
record and removes the superseded standalone design-document surface.

- Planning and design orchestration now read and write only
  `items.design_spec`.
- A governed, fail-closed migration preserves existing document bodies without
  overwriting newer structured-field content.
- Fresh universes, portability archives, fixtures, CLI help, and installation
  bundles no longer create or advertise the superseded storage surface.
