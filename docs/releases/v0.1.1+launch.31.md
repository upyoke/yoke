# Yoke 0.1.1+launch.31

## Highlights

- Make production database-admin profiles explicit so attended source-authority
  cutovers cannot silently inherit a non-production safety classification.
- Prove source credential retirement through a committed password rotation and
  successful reconnect with the cutover credential, including on managed
  PostgreSQL providers that restrict direct verifier-catalog reads.

## Upgrade notes

The normal update path is sufficient. Operators creating a database-admin
profile must now choose `--prod` or `--non-prod` explicitly.

## Compatibility

Existing connection profiles remain readable. No server API or portable
universe format changes are included.
