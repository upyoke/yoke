# Yoke 0.1.1+launch.4

## Highlights

- The installer and onboarding wizard now complete GitHub App authorization
  in the browser, preserve Back navigation, refresh repository access, and
  keep backlog-only setup first-class.
- Hosted GitHub App installation, suspension, deletion, permission, and
  repository-access events now update the exact bound project without changing
  that project's explicit sync policy.
- GitHub connection identity and repository destinations are verified against
  the configured App and selected service profile before clone, publish, or
  binding changes proceed.

## Upgrade notes

Use the normal update path. If a machine reports that an older GitHub
connection lacks App identity metadata, run `yoke github disconnect` and then
connect the GitHub App again; no token-file migration or manual credential
entry is supported.

## Compatibility

Hosted control planes that relay GitHub installation lifecycle events must run
this release or newer before enabling the corresponding Platform webhook flow.
Backlog-only projects and projects without GitHub remain supported.
