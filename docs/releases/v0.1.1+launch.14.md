# Yoke 0.1.1+launch.14

This release removes cold-host churn between consecutive self-hosted CI jobs
without converting the fleet to persistent GitHub runners.

- Each job still receives a newly registered ephemeral runner and a fresh work
  directory.
- After completion, the same EC2 host immediately rearms for another job and
  remains available until the configured idle shutdown window elapses.
- The lifecycle broker distinguishes a normal rearm window from a failed
  rearm; bounded failures still replace the host.
- Runner registration remains instance-bound and repository-scoped through
  the existing GitHub App broker.
