# Yoke 0.1.1+launch.3

## Highlights

- Local universe initialization now establishes the machine-state directory as
  owner-only before downloading embedded Postgres or creating nested runtime
  state. This keeps `yoke init --local` working on Linux distributions whose
  default umask would otherwise leave `~/.yoke` group-writable.
- Existing current-user-owned machine-state directories are tightened to mode
  `0700`; symlinked or foreign-owned directories still fail closed.

## Upgrade notes

Rerun the normal installer. Machines that encountered the local-init permission
error can then rerun `yoke init --local`; the existing partially initialized
universe is verified and completed in place.

## Compatibility

None.
