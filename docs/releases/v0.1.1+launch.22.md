# Yoke 0.1.1+launch.22

This release makes hosted machine authorization durable at the moment the
browser approval completes.

- The verified hosted credential is stored and activated before optional
  GitHub or project-onboarding steps begin, so an interrupted wizard does not
  lose an already-approved machine connection.
- A later `yoke onboard --connect https://app.upyoke.com` run reuses the saved
  tenant-scoped authority on the same hosted origin instead of issuing another
  one-time code.
- Stored hosted authorities remain origin-bound; stage, production, and
  lookalike hosts are never collapsed into one connection.

Use the normal update path. No schema migration is required.
