# Yoke 0.1.1+launch.1

## Highlights

- Yoke now installs as one complete Fair Source product: engine, CLI, harness
  adapters, skills, universe UI, and server travel together at one exact
  version.
- Onboarding defaults to a private local universe with embedded Postgres. It
  needs no account, keeps state on the machine, and supports unlimited local
  agents for one human.
- The same engine now powers local, self-hosted, and hosted deployments. Local
  users can export a portable universe archive, and `yoke self-host import`
  restores it into a fresh bundle. The restore boundary validates archive
  structure and trusted schema data, then atomically revokes imported
  platform-held API tokens and browser sessions before showing one fresh
  org-admin token.
- The packaged universe UI provides the shared read-only workbench shell, while
  the published server adds first-boot administration, OIDC sign-in, identity
  invites, and version-skew reporting.
- A bundled public GitHub App profile makes `yoke github connect` a no-flag
  machine setup flow. Installation discovery, short-lived user authorization,
  credential-helper ownership, disconnect cleanup, transport deadlines, and
  response redaction now share one bounded lifecycle.
- Release wheels and the server image are built from immutable annotated tags,
  verified against the exact source commit, and published for linux/amd64 and
  linux/arm64 with GitHub-hosted attestations.

## Upgrade notes

The normal Yoke update path is sufficient for local installations. Existing
local universes retain their durable data directory; if a machine-home path is
too long for a Unix socket, Yoke moves only the socket to a private short path
and updates the matching non-production connection automatically.

Self-host operators should back up the database, pull the image for this
release, replace the generated Compose bundle with the matching version, and
restart the stack. Schema convergence runs during server boot.

To move an existing universe into a new self-host deployment, leave `core`
stopped and run `yoke self-host import ARCHIVE --dir BUNDLE`. The destination
database must be catalog-empty and the archive must be an owner-only regular
file.

## Compatibility

- `yoke onboard` now selects local mode by default. Use `--connect URL` for a
  team server or select Yoke Cloud in the interactive destination picker.
- New project bindings start in backlog-only GitHub mode. Activate issue sync
  only after the selected App installation and repository binding verify; an
  existing explicit GitHub App profile remains supported.
- Product packages are an exact-version set; mixing wheel versions is refused.
- The HTTPS API advertises its engine version. Older clients continue to work
  across the current function-call boundary and warn when the server differs.
