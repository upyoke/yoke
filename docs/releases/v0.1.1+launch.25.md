# Yoke 0.1.1+launch.25

This release makes server image identity a build and deployment invariant.

- The shared Dockerfile rejects installed engine metadata that differs from
  the declared release, and rejects setuptools-scm's unresolved fallback when
  an image advertises a source build.
- The public multi-architecture image factory executes the pushed digest and
  verifies its installed engine version and build before attestation or tag
  publication.
- Persistent environment deploys now require the origin health response to
  serve the exact requested build with a non-empty engine version; a stale or
  metadata-incomplete container fails the release gate and rolls back.

Use the public installer again to update an existing packaged CLI. No schema
migration is required.
