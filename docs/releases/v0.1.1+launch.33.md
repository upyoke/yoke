# Yoke 0.1.1+launch.33

This release makes connection status prove the configured authority instead of
stopping at an unauthenticated health check. HTTPS connections now verify the
signed-in engine identity, while local PostgreSQL mode remains direct and
self-hosted servers use the same transport contract.

It also makes the canonical three-suite xdist gate reproducible from a fresh
checkout through the locked workspace environment and corrects the hosted
Stage and Production API authorities used by generated configuration.
