# Task 002: yoke-db.sh epic: new subcommands for all tables

## Description
Extend `yoke-db.sh epic` with CRUD subcommands covering all new tables created in task 001. This is the primary script-layer entry point for all epic data operations. Follow the existing patterns in `yoke-db.sh epic` (python3 for parameterized inserts, `_sql_query` for reads, `_escape` for simple queries).

Additionally, fix the existing `cmd_task_upsert` function. The current implementation uses `INSERT OR REPLACE` which deletes and re-inserts the row, wiping columns not in the VALUES clause (e.g., `status`, `dispatch_attempts`, `body`, `github_issue`, `branch`, `worktree_path`, `last_heartbeat`, `started_at`). Replace with `INSERT ... ON CONFLICT(epic_slug, task_num) DO UPDATE SET title=excluded.title, worktree=excluded.worktree, context_estimate=excluded.context_estimate, dependencies=excluded.dependencies` so that only the explicitly provided columns are overwritten and all other columns are preserved.

Add the following subcommands to the dispatch case at the bottom of `yoke-db.sh epic`:
- `task-update-body <epic_slug> <task_num>` (reads body from stdin)
- `task-get-body <epic_slug> <task_num>` (outputs body to stdout)
- `task-update-field <epic_slug> <task_num> <field> <value>`
- `history-insert <epic_slug> <task_num> <from_status> <to_status> [note]`
- `dispatch-chain-upsert <epic_slug> <worktree>` (reads JSON from stdin)
- `dispatch-chain-get <epic_slug> <worktree>`
- `dispatch-chain-update <epic_slug> <worktree> <field> <value>`
- `dispatch-chain-list <epic_slug>`
- `review-insert <epic_slug> <task_num> <verdict>` (reads body from stdin)
- `review-get <epic_slug> <task_num>`
- `progress-note-insert <epic_slug> <task_num> <note_num>` (reads body from stdin)
- `progress-note-list-unsynced <epic_slug>`
- `progress-note-mark-synced <epic_slug> <task_num> <note_num>`
- `simulation-upsert <epic_slug> <phase>` (reads body from stdin)
- `simulation-get <epic_slug> <phase>`
