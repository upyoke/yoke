# Task 005: update-status.sh: DB-native rewrite with backward-compatible shim

## Description
Rewrite `update-status.sh` to be DB-native while maintaining backward compatibility with the path-based interface during the transition period.

**New interface:** `update-status.sh <epic-slug> <task-num> <new-status> [note]`
**Legacy interface:** `update-status.sh <status-file-path> <new-status> [note]`

**Path-detection heuristic:** The script must detect which invocation style is used by testing `$1`. Use: `case "$1" in */status/*.json) _legacy=1 ;; *) _legacy=0 ;; esac`. If `$1` contains `/status/` and ends in `.json`, it's the legacy path-based interface — parse the path to extract `epic_slug` and `task_num`. Otherwise, treat `$1` as an epic slug and `$2` as task_num. Document this heuristic in a comment at the top of the detection block.

Core changes:
1. Add DB-native path: UPDATE `epic_tasks` + INSERT `epic_task_history` via `yoke-db.sh epic` subcommands.
2. Keep JSON file update path working in parallel (dual-write during transition).
3. Rewrite `auto_unblock()` to query `epic_tasks` instead of scanning `STATUS_DIR/*.json`.
4. Keep `rebuild_dashboard()` working but mark for removal.
5. Preserve GitHub sync, reading `github_issue` from DB.
6. Keep `update_epic_checkbox()` working with DB queries.
